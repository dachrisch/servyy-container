# Searxng Deployment Design
**Date:** 2026-06-11  
**Status:** Design Approved  
**Scope:** Deploy private meta-search engine (Google + DuckDuckGo only) via Ansible

## Overview

Deploy searxng as a managed Docker service integrated with existing infrastructure:
- **URL:** `search.lehel.xyz`
- **Scope:** Private instance with Google and DuckDuckGo engines only
- **Integration:** Traefik routing, Loki logging, Ansible deployment, Docker volumes

## Architecture

```
Internet (search.lehel.xyz)
    ↓
Traefik (auto-routes via labels)
    ↓
Docker "proxy" network
    ↓
searxng.core (search engine)
    └─ searxng.valkey (Redis cache — persisted via volume)
    └─ Promtail scrapes logs → Loki
```

### Services

**searxng.core**
- Container: `searxng/docker-compose.yml` service `core`
- Image: `docker.io/searxng/searxng:latest`
- Port: 8080 (internal), routed via Traefik to `search.lehel.xyz`
- Volumes:
  - `./core-config/` → `/etc/searxng/` (settings, mounted as non-writable)
  - `core-data:/var/cache/searxng/` (persistent)
- Network: `proxy` (for Traefik routing and DNS discovery)

**searxng.valkey**
- Container: `searxng/docker-compose.yml` service `valkey`
- Image: `docker.io/valkey/valkey:9-alpine`
- Volumes:
  - `valkey-data:/data/` (persistent)
- Network: `proxy` (internal communication)

### Data Persistence

- **core-data:** Cache storage for searxng results
- **valkey-data:** Redis/Valkey data for session/cache
- Both volumes created by docker-compose, persisted across restarts

## Configuration

### Git-Stored (Non-Secret)

**`searxng/docker-compose.yml`**
- Service definitions with Traefik labels
- Volume configuration
- Network binding
- No secrets embedded

**`searxng/core-config/settings.yml`**
```yaml
use_default_settings: true

engines:
  keep_only:
    - google
    - duckduckgo

server:
  secret_key: "{{ searxng_secret_key }}"  # Injected by Ansible
  bind_address: 0.0.0.0
  port: 8080
  image_proxy: true
```

### Ansible-Managed (Secrets)

**`.env` file (generated by Ansible role)**
- `SEARXNG_SECRET_KEY` (vault-protected)
- `SEARXNG_VERSION` (controlled variable)
- `SEARXNG_HOST` (0.0.0.0)
- `SEARXNG_PORT` (8080)

## Traefik Integration

Traefik labels in `docker-compose.yml`:
```yaml
labels:
  - "traefik.enable=true"
  - "traefik.http.routers.searxng.rule=Host(`search.lehel.xyz`)"
  - "traefik.http.routers.searxng.entrypoints=websecure"
  - "traefik.http.routers.searxng.tls.certresolver=letsencryptdnsresolver"
  - "traefik.http.services.searxng.loadbalancer.server.port=8080"
```

**Result:**
- Automatic HTTPS via Let's Encrypt
- No manual certificate management
- Automatic routing on service start/stop
- Public access via `search.lehel.xyz`

## Monitoring & Logging

**Loki Integration (Automatic)**
- Promtail discovers container automatically
- Logs scraped from stdout/stderr
- Available in Loki with label: `{job="docker",container="searxng.core"}`

**Grafana Queries:**
```
{job="docker",container="searxng.core"} | json
```

**Optional Dashboards:**
- Request latency tracking
- Engine query distribution (Google vs DuckDuckGo)
- Service health/restarts
- Error rate monitoring

No additional configuration required.

## Implementation: Ansible Role Structure

**Directory:** `ansible/plays/roles/user_searxng/`

```
user_searxng/
├── defaults/main.yml              # Default variables (image version, ports)
├── tasks/main.yml                 # Deployment tasks
├── templates/docker-compose.yml.j2 # Traefik labels, volume config
├── templates/settings.yml.j2      # Engine whitelist (Google + DuckDuckGo)
├── molecule/
│   └── default/
│       ├── converge.yml           # Test playbook
│       ├── verify.yml             # Verification tests
```

**Integration Points:**
- Role included in `ansible/plays/user.yml`
- Manages `/home/cda/servyy-container/searxng/` directory
- Generates `.env` file with secrets from vault
- Deploys via `docker-compose up -d`

## Deployment Workflow

1. **Create feature branch:** `claude/searxng-deployment`
2. **Implement Ansible role** with docker-compose and settings templates
3. **Test on servyy-test.lxd** using `./servyy-test.sh`
4. **Verify test deployment** (containers running, accessible)
5. **Request user approval** for production deployment
6. **Deploy to lehel.xyz** using `./servyy.sh --tags docker --limit lehel.xyz`
7. **Verify production** (containers running, HTTPS working, logs in Loki)
8. **Commit and document** in `history/2026-06-11_searxng-deployment.md`

## Success Criteria

- [ ] Searxng accessible via `https://search.lehel.xyz`
- [ ] Only Google and DuckDuckGo engines available
- [ ] Valkey cache persists across restarts
- [ ] HTTPS certificates auto-renewed by Traefik
- [ ] Logs appear in Loki with correct labels
- [ ] Molecule tests pass on servyy-test
- [ ] CI/CD tests pass (GitHub Actions)
- [ ] Service survives reboot (restart: always)

## Known Constraints

- Searxng runs as single instance (no clustering)
- Valkey not replicated (no HA)
- Image proxy uses searxng's built-in implementation (memory-intensive for large image searches)
- Private engine tokens currently not configured (can be added later if needed)

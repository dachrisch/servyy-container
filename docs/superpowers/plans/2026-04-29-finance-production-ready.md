## Task 1: Remove Duplicate Ofelia & Update Importer Command

**Files:**
- Modify: `finance/docker-compose.yml:91-108`

The finance compose currently defines its own Ofelia service (lines 95-108). Since Portainer already runs Ofelia globally, we remove the duplicate and rely on Portainer's instance. The importer service job labels (lines 91-93) are picked up by the centralized Ofelia.

However, the current command is a one-liner that needs to restart Ofelia after completing the import so new command changes are picked up. We'll update the command to do this.

**Current state:**
- Lines 95-108: Ofelia service definition
- Lines 91-93: Job labels on importer (these stay, but command changes)

- [ ] **Step 1: Update importer job command to restart Ofelia after import**

In the importer service labels (line 92-93), update the `daily-import.command` to:
1. Run the curl import command
2. Capture any errors
3. Restart the Ofelia container (via Docker API)

Replace lines 91-93 with:
```yaml
      - "com.ofelia.enabled=true"
      - "com.ofelia.job-exec.daily-import.schedule=0 6 * * *"
      - "com.ofelia.job-exec.daily-import.command=sh -c 'curl -X POST \"http://localhost:8080/autoupload?secret=${AUTO_IMPORT_SECRET}\" -H \"Accept: application/json\" -H \"Authorization: Bearer ${FIREFLY_III_ACCESS_TOKEN}\" -F \"json=@/import/import_config.json\" 2>&1 | tee -a /var/log/firefly-import.log && docker restart portainer.ofelia 2>&1 | tee -a /var/log/firefly-import.log'"
```

Note: The config filename changes from `import_config_2026-04-28.json` to `import_config.json` (will be generated by Ansible).

## Task 5: Add PEM File to git-crypt Encryption

**Files:**
- Modify: `.gitattributes`

Add the PEM file pattern to git-crypt so the PEM file is automatically encrypted in the repository.

- [ ] **Step 1: Add *.pem pattern to .gitattributes**

Edit `.gitattributes` and add this line:

```
*.pem filter=git-crypt diff=git-crypt
```

Full file should look like:
```
git/htpasswd filter=git-crypt diff=git-crypt
docker-compose.yml filter=git-crypt diff=git-crypt
*.yaml filter=git-crypt diff=git-crypt
secrets.* filter=git-crypt diff=git-crypt
secret_* filter=git-crypt diff=git-crypt
*.env filter=git-crypt diff=git-crypt
*.conf filter=git-crypt diff=git-crypt
*.pem filter=git-crypt diff=git-crypt
```

- [ ] **Step 2: Verify git-crypt is unlocked**

Run: `git-crypt status | head -5`

Expected: Shows "Files that are encrypted: 0" (meaning repo is unlocked) or shows encrypted file status

- [ ] **Step 3: Stage the .gitattributes change**

Run: `git add .gitattributes`

- [ ] **Step 4: Re-encrypt the PEM file with git-crypt**

The PEM file was previously unencrypted. Now that we've added the *.pem pattern, we need to re-encrypt it:

Run: `git-crypt add-gpg-user --trusted 3B6FA3E99F9ED5A4`

This re-encrypts the file (you may need to use the correct GPG key ID for your setup).

Alternatively, if git-crypt doesn't have the key, just proceed to commit and the file will be encrypted on push.

- [ ] **Step 5: Commit .gitattributes**

```bash
git commit -m "chore: add PEM files to git-crypt encryption"
```
